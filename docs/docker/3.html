도커 이미지 생성

도커에서 이미지를 빌드하기 위해서는 Dockerfile 이라는 파일을 작성해야 합니다. 이 파일에는 이미지를 구성하는 레이어를 정의하고, 필요한 패키지 및 애플리케이션을 설치하고 설정하는 등의 작업을 수행합니다.

Dockerfile 작성​

다음은 Dockerfile 간단한 예제입니다.

Dockerfile
FROM node

WORKDIR /app

COPY package.json .

RUN npm install

COPY . .

EXPOSE 3000

CMD [ "node", "app.mjs" ]

FROM
Dockerfile 시작은 보통 FROM으로 시작합니다. FROM을 통해 도커에 등록되어 있는 이미지를 쉽게 불러올 수 있습니다.

WORKDIR
WORKDIR은 도커 이미지 내부의 기본 작업 디렉터리를 설정하는 명령어입니다.

COPY
첫번 째 인자 : 컨테이너 외부 경로
두번 째 인자 : 도커 내부 경로

RUN
RUN은 명령어를 실행할 때 사용하는 명령어입니다.

EXPOSE
컨테이너로 실행 될 때 특정 포트를 방화벽을 허용하겠다는 명령어입니다.

CMD
RUN과 CMD는 둘다 실행하는 명령어지만, 가장 큰 차이점은 RUN은 이미지가 생성될 때 실행되지 않고 컨테이너로 실행될 때 실행된다는 점입니다. 또한 구문도 배열로 전달 해야 합니다.

도커 이미지 빌드​

Dockerfile을 모두 작성하셨다면 다음 명령어를 사용하여 도커 이미지로 빌드할 수 있습니다.

docker build -t my-express .

-t 옵션은 이미지에 이름을 지정하는 옵션입니다. .은 현재 디렉토리를 의미합니다.

도커 이미지 확인​

다음 명령어를 사용하여 생성된 도커 이미지 목록을 확인할 수 있습니다.

docker images

REPOSITORY   TAG        IMAGE ID       CREATED         SIZE
my-express   latest     f857fe18c68e   34 seconds ago   862MB

# -t 옵션을 사용하지 않은 경우
<none>       <none>     f857fe18c68e   34 seconds ago   862MB

도커 이미지 삭제​

다음 명령어를 사용하여 도커 이미지 삭제를 할 수 있습니다.

info

해당 이미지로 만든 컨테이너가 실행된 경우 삭제되지 않으므로, 컨테이너 먼저 중지한 후 해당 명령어 실행해야 합니다.

docker rmi `my-express` 또는 `IMAGE ID`
또는
docker image rm `my-express` 또는 `IMAGE ID`

모든 도커 이미지 삭제​

다음 명령어를 사용하여 사용하지 않는 모든 도커 이미지 삭제를 할 수 있습니다.

docker image prune
