Cypress

Cypress란 무엇인가?​

Cypress는 오픈 소스의 프론트엔드 테스팅 도구로, 웹 애플리케이션을 테스트하는 과정을 단순화하는 데 사용됩니다. Cypress는 특히 엔드 투 엔드 (End-to-End, E2E) 테스트에 중점을 둔 도구로, 웹 애플리케이션의 테스트를 작성하고 실행하는 데 있어 개발자 친화적이며 직관적인 방식을 제공합니다.

  • 단위(Unit) 테스트
  • 통합(Integation) 테스트
  • E2E(End to End) 테스트
  • 회귀(Regression) 테스트

E2E 테스트는 실제 사용자의 실행 환경과 거의 동일한 환경에서 테스트를 진행하기 때문에 실제 상황에서 발생할 수 있는 에러를 사전에 발견할 수 있습니다. 또한 수동으로 테스트할 때 시간이 많이 걸리거나, 놓칠 수 있는 부분들을 반복 테스트 하기에도 매우 유용합니다. 내가 개발한 기능 명세들이 실제 잘 동작하는지 매번 수동으로 테스트하지 않고, 자동으로 모두 정상적으로 동작한다는걸 확인할 수 있어 업무 효율성이 높아집니다.

Cypress 설정​

https://www.cypress.io

설치​

npm install cypress --save-dev

테스팅 도구 실행​

npx cypress open

cypress 폴더 구조​

 cypress
   ├── downloads # 테스트 중 파일을 다운로드 하는 경우 여기에 저장
   ├── e2e # E2E 테스트 스크립트 디렉토리
   │   └── 소유자 회원가입.cy.js
   ├── fixtures # 정적 데이터나 예제 데이터를 저장하는 디렉토리
   │   └── example.json
   └── support # 테스트 스크립트에 사용되는 보조 파일 디렉토리
       ├── commands.js
       └── e2e.js

Cypress 문법​

cy.visit(url): 테스트 진행할 URL 주소 입력

cy.visit("https://example.com");

cy.get(selector): CSS 선택자 DOM 요소를 선택

cy.get(".button-class");

cy.contains(content): 입력한 텍스트 내용을 포함하는 요소 선택

cy.contains("Login");

cy.click(): 선택된 요소 클릭

cy.get(".button-class").click();

cy.type(text): 선택된 요소 텍스트 입력

cy.get('input[type="text"]').type("example@email.com");

cy.should(conditions): 선택된 요소가 보여지는지, 특정 문구를 포함하는지 확인

cy.get(".message").should("be.visible");
cy.get(".username").should("contain", "John Doe");

cy.intercept(routeAlias, handler): 네트워크 요청을 가로채서 가짜 응답 제공 (테스트 조건에 따라 요청 제어)

cy.intercept("GET", "/api/users", { fixture: "users.json" }).as("getUsers");

cy.wait(alias): 지정된 별칭(alias)을 가진 요청이 완료될 때까지 대기

cy.wait("@getUsers");
